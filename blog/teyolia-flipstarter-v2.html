Arquitectura de Alto Rendimiento

La versión 2.0 optimiza drásticamente el flujo de trabajo comparado con sistemas monolíticos tradicionales. Al separar la lógica de contratos de la orquestación del backend, el sistema logra una robustez técnica sin precedentes en el ecosistema de eCash.

"El uso de Vite + TypeScript permite que el despliegue de una nueva campaña sea hasta un 65% más fluido para el usuario final."

El Flujo de Vida de una Campaña

Desde el borrador hasta la inmutabilidad on-chain. Un proceso diseñado para la seguridad del donante.

1

Borrador & Configuración

El creador define metas y fechas usando el wizard intuitivo en el frontend.

Local State
2

Activación Non-Custodial

Se genera el covenant. El creador firma mediante WalletConnect sin exponer su semilla.

Transaction Hex
3

Recaudación Pública

Los donantes aportan XEC. Los fondos quedan bloqueados por las reglas del pacto.

UTXO Lock
4

Resolución de Pacto

Si la meta se cumple, el creador retira. Si no, los donantes ejecutan su reembolso.

Inmutable

Integración con Tonalli

Teyolia utiliza el protocolo de WalletConnect y Deep Links para interactuar con la billetera Tonalli. Esto garantiza que el proceso de firma sea familiar para el usuario, similar a una dApp de Ethereum, pero con la eficiencia y bajos costos de eCash.

  • ✔ Soporte nativo para dispositivos móviles.
  • ✔ Validación de red automática.
  • ✔ Manejo de estados de carga asíncronos.
